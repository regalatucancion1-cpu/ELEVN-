 # ELEVN DJ'S — TEST DE MATCHING

## Objetivo del Test

El test NO asigna un DJ específico.
El test identifica el **perfil musical** que encaja con la boda.
ELEVN decide el DJ final basándose en disponibilidad y criterio.

---

## Lógica del Matching

| Perfil | Señales típicas |
|--------|-----------------|
| **Club Sophisticate** | Quieren elegancia, mencionan "club", "house", "disco", prefieren atmósfera sobre hits |
| **Crossover Elegant** | Quieren "un poco de todo", crowd mixto, equilibrio entre conocido y buen gusto |
| **High Energy Driver** | Quieren fiesta fuerte, "everyone dancing", energía alta, final épico |
| **Global Urban Selector** | Mencionan R&B, hip-hop, Afrobeats, crowd diverso/internacional |

---

## Estructura del Test

### Sección 2: VIBE Y ATMÓSFERA (puntúa)

# SCORING SYSTEM

## Cómo calcular el perfil

Suma los puntos de cada perfil basándote en las respuestas.

| Perfil | Si puntuación más alta |
|--------|------------------------|
| Club Sophisticate | Asignar perfil CS |
| Crossover Elegant | Asignar perfil CE |
| High Energy Driver | Asignar perfil HED |
| Global Urban Selector | Asignar perfil GUS |

### En caso de empate:
1. Revisar pregunta 12 (descripción de fiesta que les gustó)
2. Considerar composición del crowd (pregunta 9)
3. Si sigue sin ser claro: asignar **Crossover Elegant** como perfil seguro

# NOTAS FINALES

## El test NO es definitivo

El test da una guía. Tú decides el DJ final basándote en:
- Disponibilidad real
- Criterio propio
- Detalles que el test no captura

## El cliente NO ve el perfil técnico

El cliente recibe: "We've matched you with our [Profile Name] sound"
No explicamos la lógica interna.

## El test puede evolucionar

Después de 10-20 usos, revisa:
- ¿Las asignaciones tienen sentido?
- ¿Hay preguntas que no aportan?
- ¿Falta alguna pregunta clave?

Ajusta según datos reales.
